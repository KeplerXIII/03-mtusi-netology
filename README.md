# enterprise-ai-knowledge-platform
Enterprise AI Knowledge Platform — система корпоративного поиска и интеллектуального анализа документов, объединяющая семантический поиск (RAG) и большие языковые модели с дообучением (LoRA) для извлечения, обработки и интерпретации знаний с целью поддержки принятия решений.


---

# Экономика решения

## 1. Сценарий использования

* Пользователи: **100**
* Среднее число запросов на пользователя: **20 / день**
* Всего запросов: **2000 / день (~60 000 / месяц)**

Параметры запроса:

* Средний input: **1500 токенов** (RAG + вопрос)
* Средний output: **500 токенов**
* Итого на запрос: **2000 токенов**

Общий объём:

* В день: **4 млн токенов**
* В месяц: **120 млн токенов**

---

## 2. Стоимость облачных моделей

### Флагман (GPT-5.4 / Claude Opus)

Средняя цена:

* input: ~$15 / 1M токенов
* output: ~$60 / 1M токенов

Расчёт:

* input: 90M токенов → $1350
* output: 30M токенов → $1800
  **Итого: ~$3150 / месяц**

---

### Средний сегмент (Gemini Pro / Claude Sonnet)

Средняя цена:

* input: ~$3 / 1M
* output: ~$15 / 1M

Расчёт:

* input: 90M → $270
* output: 30M → $450
  **Итого: ~$720 / месяц**

---

### Бюджет (DeepSeek V3 / GPT-5 mini)

Средняя цена:

* input: ~$0.5 / 1M
* output: ~$2 / 1M

Расчёт:

* input: 90M → $45
* output: 30M → $60
  **Итого: ~$105 / месяц**

---

## 3. Локальное развёртывание

Конфигурация:

* 4×A100 80GB (или аналог)
* CPU + RAM + storage
* Сервер: ~$120 000
* Амортизация: 3 года

Расчёт:

* $120 000 / 36 мес ≈ **$3330 / месяц**
* Электроэнергия + охлаждение: ~$300
* Обслуживание: ~$300

**Итого: ~$3900 / месяц**

---

## 4. Точка безубыточности

Сравнение:

| Вариант          | Стоимость |
| ---------------- | --------- |
| Облако (бюджет)  | $105      |
| Облако (средний) | $720      |
| Облако (флагман) | $3150     |
| Локально         | $3900     |

Вывод:

* Локальное решение становится выгодным при нагрузке **>150–200 млн токенов/месяц**
* При текущем сценарии дешевле облако

---

## 5. Выбор стратегии

Рекомендуется **гибридная модель**:

* Бюджетные запросы → локальная модель (Qwen / DeepSeek)
* Критические задачи → облачные модели (GPT / Claude)
* RAG и embeddings → локально

Обоснование:

* Минимизация стоимости при сохранении качества
* Контроль данных внутри периметра
* Масштабируемость без капитальных затрат на старте
* Возможность постепенного перехода к on-prem при росте нагрузки
